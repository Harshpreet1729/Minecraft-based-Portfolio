# Build an Advanced Minecraft Inspired Developer Portfolio

You are an expert frontend engineer, creative director, UI designer, motion designer, and Minecraft style world builder.

Your task is to create a highly polished, modern, recruiter friendly, Minecraft inspired developer portfolio. This should not look like a normal portfolio with Minecraft textures randomly added. It should feel like a custom playable Minecraft style world built around my developer journey.

Use modern frontend tools, not plain HTML CSS JS only.

Recommended stack:

Use Vite with React and TypeScript.

Use Tailwind CSS for fast styling and responsive layout.

Use Framer Motion for smooth page transitions, hover animations, item popups, modal transitions, and advancement effects.

Use React Router for separate pages.

Use lightweight 3D or pseudo 3D only where it improves the design. You may use Three.js or React Three Fiber only for small controlled hero visuals, not for the whole website.

Use CSS modules or clean component based styling where needed.

Keep the code clean, modular, scalable, and easy to understand.

## Asset Location

Minecraft assets, textures, icons, and font files are available at:

`D:\minecraft-Portfolio\minecraft _textures`

Use this folder as the main source for visual assets.

There is also a Minecraft style `.ttf` font inside this folder. Find it and load it using `@font-face`.

Use this Minecraft style font across the whole website, including headings, buttons, labels, cards, navigation, tooltips, and body text. Since Minecraft fonts can be hard to read, improve readability using proper letter spacing, line height, font size, contrast, and spacing.

Use the provided textures creatively:

Wood textures for signs, buttons, panels, and project frames.

Stone textures for section backgrounds and borders.

Grass and dirt textures for the homepage spawn area.

Diamond and emerald textures for premium buttons, highlighted projects, and important calls to action.

Obsidian textures for dark panels, footer, and portal inspired sections.

Redstone textures for backend, logic, APIs, and interactive links.

Book textures for resume, blogs, and project documentation.

Chest textures for GitHub, storage, database, and code repository sections.

Avoid stretching textures badly. Use background sizing, pixel rendering, repeat patterns, and layered overlays carefully.
